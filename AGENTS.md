## 1. 공통 기반 규칙

모든 레이아웃이 공유하는 유니코드 조합 공식, 자모 인덱스, 복합 자모 테이블을 정의한다.

### 1-1. 유니코드 음절 조합 공식

```
완성형 음절 코드포인트 = 0xAC00 + (초성_idx × 21 + 중성_idx) × 28 + 종성_idx
```

- 유효 범위: `0xAC00`(가) ~ `0xD7A3`(힣)
- 종성 없음 = `종성_idx 0`

### 1-7. 공통 FSM 상태 정의

모든 레이아웃이 동일한 FSM 상태를 공유한다.

| 상태 ID  | 이름              | 설명                          | 예시 |
|----------|-------------------|-------------------------------|------|
| `S0`     | EMPTY             | 버퍼 비어있음                 |      |
| `S1`     | CHO               | 초성만 입력됨                 | ㄱ   |
| `S2`     | CHO_JUNG          | 초성+중성                     | 가   |
| `S3`     | CHO_JUNG_JONG     | 초성+중성+홑받침              | 각   |
| `S3D`    | CHO_JUNG_JONG2    | 초성+중성+겹받침              | 닭   |
| `S4`     | JUNG_ONLY         | 중성만 (모음 시작)            | ㅏ   |

> **current**: 현재 조합 중인 글자 (화면에 밑줄로 표시)  
> **committed**: 확정된 문자열 버퍼

### 1-8. 공통 FSM 전이 규칙

#### S0 (EMPTY)

| 입력    | 다음 상태 | 동작                                  |
|---------|-----------|---------------------------------------|
| 자음 C  | S1        | current = C                           |
| 모음 V  | S4        | current = V                           |
| BS      | S0        | committed 마지막 문자 삭제 (없으면 무시) |
| Space   | S0        | commit(' ')                           |
| 기타    | S0        | commit(해당 문자)                     |

#### S1 (CHO)

| 입력    | 다음 상태 | 동작                                          |
|---------|-----------|-----------------------------------------------|
| 모음 V  | S2        | current = compose(C, V)                       |
| 자음 C2 | S1        | commit(C); current = C2                       |
| BS      | S0        | current = 없음                                |
| Space   | S0        | commit(C); commit(' ')                        |
| 기타    | S0        | commit(C); commit(기타)                       |

#### S2 (CHO_JUNG)

| 입력              | 다음 상태 | 동작                                                              |
|-------------------|-----------|-------------------------------------------------------------------|
| 모음 V2 (복합가능) | S2        | current = compose(초성, compound(중성, V2))                       |
| 모음 V2 (복합불가) | S2        | commit(current); current = compose(ㅇ, V2)                       |
| 자음 C            | S3        | current = compose(초성, 중성, C)                                  |
| BS                | S1        | current = 초성 (중성 제거)                                        |
| Space             | S0        | commit(current); commit(' ')                                      |
| 기타              | S0        | commit(current); commit(기타)                                     |

#### S3 (CHO_JUNG_JONG)

| 입력              | 다음 상태 | 동작                                                              |
|-------------------|-----------|-------------------------------------------------------------------|
| 모음 V            | S2        | commit(compose(초성, 중성)); current = compose(종성, V)           |
| 자음 C2 (겹받침)  | S3D       | current = compose(초성, 중성, double(종성, C2))                   |
| 자음 C2 (겹받침X) | S1        | commit(current); current = C2                                     |
| BS                | S2        | current = compose(초성, 중성) (종성 제거)                         |
| Space             | S0        | commit(current); commit(' ')                                      |
| 기타              | S0        | commit(current); commit(기타)                                     |

#### S3D (CHO_JUNG_JONG2 — 겹받침)

| 입력    | 다음 상태 | 동작                                                                       |
|---------|-----------|----------------------------------------------------------------------------|
| 모음 V  | S2        | commit(compose(초성, 중성, 겹받침_왼쪽)); current = compose(겹받침_오른쪽, V) |
| 자음 C  | S1        | commit(current); current = C                                               |
| BS      | S3        | current = compose(초성, 중성, 겹받침_왼쪽) (겹받침 → 홑받침)              |
| Space   | S0        | commit(current); commit(' ')                                               |
| 기타    | S0        | commit(current); commit(기타)                                              |

#### S4 (JUNG_ONLY)

| 입력              | 다음 상태 | 동작                                                |
|-------------------|-----------|-----------------------------------------------------|
| 모음 V2 (복합가능) | S4        | current = compound(현재모음, V2)                    |
| 모음 V2 (복합불가) | S4        | commit(current); current = V2                       |
| 자음 C            | S1        | commit(current); current = C                        |
| BS                | S0        | current = 없음                                      |
| Space             | S0        | commit(current); commit(' ')                        |
| 기타              | S0        | commit(current); commit(기타)                       |

> **S4 Backspace 예외**: 복합 중성 조합 중이었다면 첫 번째 모음으로만 되돌린다.  
> 예) `ㅘ` → BS → `ㅗ`

### 1-9. 공통 특수 처리

| 이벤트           | 처리                                      |
|------------------|-------------------------------------------|
| 커서 이동        | commit(current); S0 (강제 확정)           |
| 포커스 아웃      | commit(current); S0 (강제 확정)           |
| 한영 전환        | commit(current); S0 → 영문 모드           |
| Enter            | commit(current); commit('\n'); S0         |

### 1-10. Backspace 동작 명세

Backspace 동작은 두 가지 경우로 나뉜다.

**원칙:**
- **글자가 완성된 상태(S0)**: committed 마지막 글자 1개를 통째로 삭제한다.
- **글자가 조합 중인 상태(S1~S4)**: FSM 상태를 한 단계 뒤로 되돌린다 (초성→중성→종성 단위).

#### 1-10-1. FSM 상태별 Backspace

| FSM 상태        | BS 후 상태 | 동작                               | 예시        |
|-----------------|------------|------------------------------------|-------------|
| S0 (확정 있음)  | S0         | committed 마지막 글자 1개 삭제     | 한글 → 한   |
| S0 (확정 없음)  | S0         | 무시                               | (빈) → 무시 |
| S1              | S0         | 초성 제거                          | ㄱ → (빈)   |
| S2              | S1         | 중성 제거                          | 가 → ㄱ     |
| S2 (복합 중성)  | S2         | 복합 중성 → 첫 번째 단모음         | 과 → 고     |
| S3              | S2         | 종성 제거                          | 각 → 가     |
| S3D             | S3         | 겹받침 → 홑받침                    | 닭 → 달     |
| S4              | S0         | 단독 모음 제거                     | ㅏ → (빈)   |
| S4 (복합 중성)  | S4         | 복합 중성 → 첫 번째 단모음         | ㅘ → ㅗ     |

> **S0 BS 구현 주의**: 오토마타가 committed를 직접 관리하는 경우 마지막 유니코드 코드포인트 1개를 제거한다. 텍스트 필드에 위임하는 구조라면 `deletePreviousChar()` 호출로 처리한다.

#### 1-10-2. 레이아웃별 Backspace 처리 위임 원칙

멀티탭(천지인·SKY-II·모토로라·나랏글)이나 전처리 컨텍스트(단모음 이중모음·천지인 VowelContext)가 있는 레이아웃에서도 **Backspace는 FSM 상태 기준으로만 동작한다.**

- 멀티탭 진행 중이더라도 **모음이 FSM에 전달된 이후**에는 공통 FSM BS 규칙을 따른다.
- 레이아웃 전처리 단계(예: 타임아웃 전, 모음 조합 버퍼)에서 BS가 발생하면 해당 전처리 컨텍스트를 초기화하고 FSM에는 영향을 주지 않는다.

```
예) SKY-II에서 `1`×3(ㄲ)이 FSM에 전달된 후 ㅏ 입력 = "까"
    BS → ㄲ(S1)    ← FSM 기준, 중성만 제거
    BS → (빈, S0)  ← FSM 기준, 초성 제거
    (ㄲ을 ㅋ→ㄱ으로 역분해하지 않음)

예) 천지인에서 ㅣ + ㆍ = ㅏ가 FSM에 전달된 후 종성 입력 = "안"
    BS → 아(S2)  ← FSM 기준, 종성 제거
    BS → ㅇ(S1)  ← FSM 기준, 중성 제거
    BS → (빈)    ← FSM 기준, 초성 제거
```

## 2. 두벌식 (Dubeolsik)

### 2-1. 키 레이아웃

```
[ㅂ][ㅈ][ㄷ][ㄱ][ㅅ][ㅛ][ㅕ][ㅑ][ㅐ][ㅔ]
[ㅁ][ㄴ][ㅇ][ㄹ][ㅎ][ㅗ][ㅓ][ㅏ][ㅣ]
```

### 2-3. 자모 분류

```
자음(초성/종성 가능) = {ㄱ,ㄲ,ㄴ,ㄷ,ㄸ,ㄹ,ㅁ,ㅂ,ㅃ,ㅅ,ㅆ,ㅇ,ㅈ,ㅉ,ㅊ,ㅋ,ㅌ,ㅍ,ㅎ}
모음(중성 가능)      = {ㅏ,ㅐ,ㅑ,ㅒ,ㅓ,ㅔ,ㅕ,ㅖ,ㅗ,ㅛ,ㅜ,ㅠ,ㅡ,ㅣ}
```

> 쌍자음(ㄸ, ㅃ, ㅉ)은 초성에만 사용 가능하며 종성에 올 수 없다.  
> ㄲ, ㅆ은 초성과 종성 모두 사용 가능하다.

### 2-4. FSM

두벌식은 §1.7 ~ §1.8의 공통 FSM을 **그대로** 사용한다. 레이아웃 고유 처리 없음.

### 2-5. 검증 케이스

| 입력 시퀀스 (키)        | 기대 출력   | 설명                          |
|-------------------------|-------------|-------------------------------|
| `r` `k`                 | 가          | 초성+중성                     |
| `r` `k` `s`             | 간          | 초성+중성+종성                |
| `r` `k` `s` `k`         | 가나         | 종성→다음초성 이동            |
| `e` `k` `f` `r`         | 닭 (조합중) | ㄷ+ㅏ+ㄹ+ㄱ = 겹받침 ㄺ      |
| `e` `k` `f` `r` `k`     | 달가         | 겹받침 분리 후 다음 글자      |
| `d` `h` `k`             | 와          | ㅗ+ㅏ 복합중성                |
| `d` `h` `o`             | 왜          | ㅗ+ㅐ = ㅙ                    |
| `r` `k` `s` → BS        | 가          | 종성 제거                     |
| `r` `k` → BS            | ㄱ (S1)     | 중성 제거                     |
| `r` → BS                | (빈)        | 초성 제거                     |
| `e` `k` `f` `r` → BS   | 달          | 겹받침 → 홑받침               |
| `e` `k` `f` `r` → BS → BS | 다       | 홑받침 제거                   |

## 3. 단모음 (Danmoeum)

### 3-1. 개요

두벌식의 직접적인 변형 레이아웃이다. `⇧`(Shift) 키 없이 **동일 키 두 번 입력**으로 쌍자음·이중모음을 생성하는 것이 핵심이다.

- **자음**: 같은 자음 키 두 번 → 쌍자음 (예: `ㅂ``ㅂ` → ㅃ)
- **모음**: 같은 모음 키 두 번 → 이중모음 (예: `ㅏ``ㅏ` → ㅑ)
- **복합 중성(ㅘ 등)**: §1.5 테이블 그대로 적용 (두벌식과 동일)
- **FSM**: 두벌식 FSM(§1.7~§1.8)과 완전히 동일, 키 해석 전처리만 다름

### 3-2. 키 레이아웃

```
[ㅂ][ㅈ][ㄷ][ㄱ][ㅅ][ㅗ][ㅐ][ㅔ]
[ㅁ][ㄴ][ㅇ][ㄹ][ㅎ][ㅓ][ㅏ][ㅣ]
[ㅋ][ㅌ][ㅊ][ㅍ][ㅜ][ㅡ]
```

### 3-4. 동일 키 두 번 입력 변환 테이블

FSM에 자모를 전달하기 **전**, 아래 전처리 규칙을 적용한다.  
직전 입력과 현재 입력이 동일한 키일 때 두 입력을 합쳐 변환된 자모 하나로 교체한다.

#### 쌍자음 변환 (자음 키 두 번)

| 키 (×2) | 단독 자모 | 변환 결과 |
|---------|-----------|-----------|
| `Q``Q`  | ㅂ+ㅂ     | ㅃ        |
| `W``W`  | ㅈ+ㅈ     | ㅉ        |
| `E``E`  | ㄷ+ㄷ     | ㄸ        |
| `R``R`  | ㄱ+ㄱ     | ㄲ        |
| `T``T`  | ㅅ+ㅅ     | ㅆ        |

> ㅁ, ㄴ, ㅇ, ㄹ, ㅎ, ㅋ, ㅌ, ㅊ, ㅍ은 쌍자음 없음 — 두 번 입력해도 각각 독립 자음으로 처리.

#### 이중모음 변환 (모음 키 두 번)

| 키 (×2) | 단독 모음 | 변환 결과 |
|---------|-----------|-----------|
| `J``J`  | ㅏ+ㅏ     | ㅑ        |
| `H``H`  | ㅓ+ㅓ     | ㅕ        |
| `Y``Y`  | ㅗ+ㅗ     | ㅛ        |
| `B``B`  | ㅜ+ㅜ     | ㅠ        |
| `U``U`  | ㅐ+ㅐ     | ㅒ        |
| `I``I`  | ㅔ+ㅔ     | ㅖ        |

> ㅣ(`K`), ㅡ(`N`)는 이중모음 없음 — 두 번 입력하면 각각 독립 모음으로 처리.

### 3-5. 전처리 규칙 상세

```
전처리 함수 preprocess(prevKey, currKey) -> KeyInput:

  IF prevKey == currKey:
    combined = doubleMap[currKey]   // §3.4 테이블 참조
    IF combined != null:
      RETURN KeyInput(combined)     // 두 입력을 하나의 변환된 자모로 교체
                                    // prevKey 소비됨 (버퍼에서 제거)
    ELSE:
      RETURN KeyInput(keyMap[currKey])  // 변환 없음, 두 번째 입력을 독립 자모로 처리
  ELSE:
    RETURN KeyInput(keyMap[currKey])    // 일반 처리
```

**중요한 경계 케이스:**

1. **쌍자음 + 모음**: `R``R``J` → ㄲ(쌍자음 확정) + ㅏ → "까"
2. **모음 두 번 + 다른 모음**: `J``J``H` → ㅑ 확정 후 ㅓ 시작
3. **같은 키 세 번**: `R``R``R` → ㄲ(소비) + ㄱ(새 입력) → ㄲ 확정 후 ㄱ 시작
4. **쌍자음 불가 자음 두 번**: `S``S` → ㄴ 확정 후 ㄴ 새 초성
5. **이중모음 불가 모음 두 번**: `K``K` → ㅣ 확정 후 ㅣ 새 글자 시작

### 3-6. 두벌식과의 차이 요약

| 항목           | 두벌식                              | 단모음                                   |
|----------------|-------------------------------------|------------------------------------------|
| FSM            | §1.7~§1.8                           | **동일** (§1.7~§1.8)                     |
| 복합 중성      | 지원                                | **동일하게 지원**                        |
| 겹받침         | 지원                                | **동일하게 지원**                        |
| 쌍자음 입력    | `⇧`+키                             | **동일 자음 키 두 번** (`Q``Q` → ㅃ)    |
| 이중모음 입력  | 단독 키 (ㅑ·ㅕ·ㅛ·ㅠ 별도 키 존재) | **동일 모음 키 두 번** (`J``J` → ㅑ)   |
| 전처리         | 없음                                | **동일 키 두 번 감지 후 변환**           |

## 4. 천지인 (Cheonjiin)

### 4-1. 개요

삼성전자 고유 방식. 사람(ㅣ), 하늘(ㆍ), 땅(ㅡ) 세 기본 모음 획을 조합하여 모든 모음을 만든다. 자음은 기본자에 동일 키 반복 입력으로 파생자를 만든다.

### 4-2. 키 레이아웃

```
[ ㅣ ][ ㆍ ][ ㅡ ]
[ㄱㅋ][ㄴㄹ][ㄷㅌ]
[ㅂㅍ][ㅅㅎ][ㅈㅊ]
[ ?? ][ㅇㅁ][ ?? ]
```

| 키  | 할당         | 비고                          |
|-----|--------------|-------------------------------|
| `1` | ㅣ           | 인(人) — 모음 기본획          |
| `2` | ㆍ           | 천(天) — 모음 기본획          |
| `3` | ㅡ           | 지(地) — 모음 기본획          |
| `4` | ㄱ / ㅋ      | 자음 (반복 입력으로 파생)     |
| `5` | ㄴ / ㄹ      | 자음                          |
| `6` | ㄷ / ㅌ      | 자음                          |
| `7` | ㅂ / ㅍ      | 자음                          |
| `8` | ㅅ / ㅎ      | 자음                          |
| `9` | ㅈ / ㅊ      | 자음                          |
| `0` | ㅇ / ㅁ      | 자음                          |
| `*` | 구현에 따라 다름 | 쌍자음, 기호, 스페이스 등  |
| `#` | 구현에 따라 다름 | 확정, 모드전환 등             |

> `*`, `#` 동작은 제조사/앱 구현에 따라 다르므로 구현 시 주입(inject) 가능하도록 설계할 것.

### 4-3. 자음 파생 규칙 (동일 키 반복 입력)

| 키  | 1회 | 2회 | 3회 |
|-----|-----|-----|-----|
| `4` | ㄱ  | ㅋ  | ㄲ  |
| `5` | ㄴ  | ㄹ  | (없음) |
| `6` | ㄷ  | ㅌ  | ㄸ  |
| `7` | ㅂ  | ㅍ  | ㅃ  |
| `8` | ㅅ  | ㅎ  | ㅆ  |
| `9` | ㅈ  | ㅊ  | ㅉ  |
| `0` | ㅇ  | ㅁ  | (없음) |

> **쌍자음 입력**: `4`, `6`, `7`, `8`, `9` 키를 **3회** 연속 입력하면 된소리(쌍자음)가 나온다.  
> **예외**: `5`(ㄴㄹ)와 `0`(ㅇㅁ)은 쌍자음이 없으므로 3회 입력 시 **1회 상태(기본자)로 순환**한다.  
> 타임아웃(기본 1000ms) 또는 다른 키 입력 시 현재 자모 확정.

### 4-4. 모음 조합 규칙

천지인 모음은 `ㅣ`(1번), `ㆍ`(2번), `ㅡ`(3번) 세 키의 **순서 있는 조합**으로 생성된다.

> `ㆍ`(가운뎃점, U+318D)는 단독으로는 문자로 출력되지 않으며 오직 모음 조합에만 사용된다.

#### 완성 모음 생성 테이블

| 입력 시퀀스          | 결과 모음 | 설명                    |
|----------------------|-----------|-------------------------|
| `ㅣ`                 | ㅣ        | 단독                    |
| `ㅡ`                 | ㅡ        | 단독                    |
| `ㅣ` `ㆍ`            | ㅏ        | ㅣ 오른쪽에 점 → ㅏ     |
| `ㆍ` `ㅣ`            | ㅓ        | ㅣ 왼쪽에 점 → ㅓ       |
| `ㅣ` `ㆍ` `ㆍ`       | ㅑ        | ㅣ 오른쪽에 점×2 → ㅑ   |
| `ㆍ` `ㆍ` `ㅣ`       | ㅕ        | ㅣ 왼쪽에 점×2 → ㅕ     |
| `ㆍ` `ㅡ`            | ㅗ        | ㅡ 위에 점 → ㅗ         |
| `ㅡ` `ㆍ`            | ㅜ        | ㅡ 아래에 점 → ㅜ       |
| `ㆍ` `ㆍ` `ㅡ`       | ㅛ        | ㅡ 위에 점×2 → ㅛ       |
| `ㅡ` `ㆍ` `ㆍ`       | ㅠ        | ㅡ 아래에 점×2 → ㅠ     |
| `ㅣ` `ㆍ` `ㅣ`       | ㅐ        | ㅏ + ㅣ 추가            |
| `ㆍ` `ㅣ` `ㅣ`       | ㅔ        | ㅓ + ㅣ 추가            |
| `ㅣ` `ㆍ` `ㆍ` `ㅣ`  | ㅒ        | ㅑ + ㅣ 추가            |
| `ㆍ` `ㆍ` `ㅣ` `ㅣ`  | ㅖ        | ㅕ + ㅣ 추가            |
| `ㅡ` `ㅣ`            | ㅢ        | 땅+사람                 |

#### 복합 중성 (ㅘ 계열)

천지인에서 복합 중성은 **완성된 모음 두 개를 연속 입력**하여 §1.5 테이블로 조합한다.  
예) ㅗ(`ㆍ``ㅡ` 완성) → ㅏ(`ㆍ``ㅣ` 완성) → §1.5: ㅗ+ㅏ=ㅘ

### 4-5. 모음 조합 컨텍스트 (VowelContext)

FSM에 모음 이벤트를 전달하기 전, 별도의 VowelContext에서 `ㅣ`/`ㆍ`/`ㅡ` 시퀀스를 누적한다.

```
VowelContext 처리 흐름:

  입력: ㅣ, ㆍ, ㅡ 중 하나
    → vowelBuffer에 추가
    → vowelBuffer가 §4.4 테이블의 완성 패턴과 일치하는가?
        YES: 완성 모음을 FSM에 전달, vowelBuffer 초기화
        NO:  더 긴 패턴의 접두사인가?
               YES: 대기 (다음 입력 기다림)
               NO:  vowelBuffer 첫 번째 항목으로 가능한 최장 일치 모음 확정
                    나머지를 새 vowelBuffer로 재시도

  입력: 자음 키
    → vowelBuffer가 비어있지 않으면: 현재까지 누적된 패턴으로 가능한 모음 확정
    → 자음 처리로 전환

  Backspace:
    → vowelBuffer가 비어있지 않으면: vowelBuffer 마지막 항목 제거 후 재평가
    → 비어있으면: FSM Backspace 처리
```

#### 미완성 패턴 처리 예시

- `ㆍ` 단독 입력 후 자음 → `ㆍ`는 어떤 완성 모음도 아님 → 버리고 자음 처리
- `ㆍ` `ㆍ` 입력 후 자음 → 마찬가지로 버리고 자음 처리
- `ㆍ` `ㅣ` → ㅏ 완성 → FSM에 ㅏ 전달

### 4-6. FSM

공통 FSM(§1.7~§1.8)을 그대로 사용한다.  
VowelContext가 완성된 모음을 FSM에 전달하면, FSM은 해당 모음을 일반 모음 입력으로 처리한다.

| `4` `ㅣ` `ㆍ` `5` → BS            | 가 (S2)    | 종성 ㄴ 제거                           |

## 5. KT 나랏글 (Naratgeul)

### 5-1. 개요

KT(구 한국통신)가 개발한 방식. 자음 6개와 모음 4개 키로 구성된 3×4 키패드 레이아웃이다.  
`획추가`(`*`) 키가 **자음과 모음 모두에** 적용되어 파생 자모를 만든다.

### 5-2. 키 레이아웃

```
[ ㄱ ][ ㄴ ][ㅏㅓ]
[ ㄹ ][ ㅁ ][ㅗㅜ]
[ ㅅ ][ ㅇ ][ ㅣ ]
[획추가][ㅡ][쌍자음]
```

| 키       | 할당     | 키 ID |
|----------|----------|-------|
| `ㄱ`     | ㄱ 계열  | `1`   |
| `ㄴ`     | ㄴ 계열  | `2`   |
| `ㅏㅓ`   | ㅏ / ㅓ  | `3`   |
| `ㄹ`     | ㄹ       | `4`   |
| `ㅁ`     | ㅁ 계열  | `5`   |
| `ㅗㅜ`   | ㅗ / ㅜ  | `6`   |
| `ㅅ`     | ㅅ 계열  | `7`   |
| `ㅇ`     | ㅇ 계열  | `8`   |
| `ㅣ`     | ㅣ       | `9`   |
| `획추가` | 획 추가  | `*`   |
| `ㅡ`     | ㅡ       | `0`   |
| `쌍자음` | 쌍자음   | `#`   |

### 5-3. 자음 파생 규칙

자음 키 입력 후 `획추가`(`*`)를 누르면 획이 추가된 파생 자음으로 교체된다. `획추가` 반복 시 순환한다.

| 키  | 기본 | `*` 1회 | `*` 2회 | `*` 3회 |
|-----|------|---------|---------|---------|
| `1` | ㄱ   | ㅋ      | (순환)  |         |
| `2` | ㄴ   | ㄷ      | ㅌ      | (순환)  |
| `4` | ㄹ   | (없음)  |         |         |
| `5` | ㅁ   | ㅂ      | ㅍ      | (순환)  |
| `7` | ㅅ   | ㅈ      | ㅊ      | (순환)  |
| `8` | ㅇ   | ㅎ      | (순환)  |         |

> `획추가`는 직전에 입력된 **자음 또는 모음**이 조합 중인 상태에서만 유효하다.  
> 자음 확정(모음 입력 등) 이후 `획추가` 입력 시 무효 처리된다.

### 5-4. 모음 입력 규칙

모음 키 `3`(ㅏㅓ), `6`(ㅗㅜ)은 **1회=기본 모음, 2회=파생 모음**으로 구분한다.  
`획추가`(`*`)로 이중 모음(ㅑ·ㅕ·ㅛ·ㅠ)을 추가 파생한다.

#### 기본 모음 (키 단독 입력)

| 키  | 1회 | 2회 |
|-----|-----|-----|
| `3` | ㅏ  | ㅓ  |
| `6` | ㅗ  | ㅜ  |
| `9` | ㅣ  | (없음) |
| `0` | ㅡ  | (없음) |

> `3`키 2회 입력 = ㅓ. 타임아웃(기본 1000ms) 또는 다른 키 입력 시 현재 모음 확정.

#### `획추가`로 이중 모음 파생

확정되지 않은 모음 상태에서 `획추가`(`*`)를 누르면 이중 모음으로 교체된다.

| 현재 모음 | `*` 입력 후 | 설명         |
|-----------|-------------|--------------|
| ㅏ        | ㅑ          | ㅏ + 획추가  |
| ㅓ        | ㅕ          | ㅓ + 획추가  |
| ㅗ        | ㅛ          | ㅗ + 획추가  |
| ㅜ        | ㅠ          | ㅜ + 획추가  |

> ㅣ, ㅡ는 획추가 이중 모음 없음.

#### 모음 입력 전체 흐름 요약

```
ㅏ 입력:  `3`×1
ㅓ 입력:  `3`×2
ㅑ 입력:  `3`×1 + `*`       (ㅏ + 획추가)
ㅕ 입력:  `3`×2 + `*`       (ㅓ + 획추가)
ㅗ 입력:  `6`×1
ㅜ 입력:  `6`×2
ㅛ 입력:  `6`×1 + `*`       (ㅗ + 획추가)
ㅠ 입력:  `6`×2 + `*`       (ㅜ + 획추가)
ㅣ 입력:  `9`×1
ㅡ 입력:  `0`×1
```

### 5-5. 복합 중성 조합

기본/이중 모음 확정 후 추가 모음 입력으로 복합 중성을 생성한다. §1.5 복합 중성 테이블과 동일하게 적용된다.

| 입력 시퀀스               | 결과 모음 | 설명                      |
|---------------------------|-----------|---------------------------|
| `3`×1, `9`               | ㅐ        | ㅏ + ㅣ                   |
| `3`×2, `9`               | ㅔ        | ㅓ + ㅣ                   |
| `3`×1, `*`, `9`          | ㅒ        | ㅑ + ㅣ (ㅐ에 이중 적용)  |
| `3`×2, `*`, `9`          | ㅖ        | ㅕ + ㅣ                   |
| `6`×1, `3`×1             | ㅘ        | ㅗ + ㅏ                   |
| `6`×1, `3`×1, `9`        | ㅙ        | ㅗ + ㅏ + ㅣ              |
| `6`×1, `9`               | ㅚ        | ㅗ + ㅣ                   |
| `6`×2, `3`×2             | ㅝ        | ㅜ + ㅓ                   |
| `6`×2, `3`×2, `9`        | ㅞ        | ㅜ + ㅓ + ㅣ              |
| `6`×2, `9`               | ㅟ        | ㅜ + ㅣ                   |
| `0`, `9`                  | ㅢ        | ㅡ + ㅣ                   |

### 5-6. 쌍자음 생성 (`#` 키)

자음이 조합 중인 상태에서 `쌍자음`(`#`) 입력 시 아래 변환:

| 원래 | → | 쌍자음 |
|------|---|--------|
| ㄱ   | → | ㄲ     |
| ㄷ   | → | ㄸ     |
| ㅂ   | → | ㅃ     |
| ㅅ   | → | ㅆ     |
| ㅈ   | → | ㅉ     |

> 파생 자음(예: ㅅ→ㅈ 후 `#` → ㅉ)에도 적용 가능하다.

### 5-7. `획추가` 적용 우선순위

`획추가`(`*`)는 직전 자모의 종류에 따라 다르게 동작한다:

| 직전 자모 상태     | `*` 동작                     |
|--------------------|------------------------------|
| 자음 조합 중       | 자음 파생 (§5.3)             |
| 모음 조합 중       | 이중 모음 파생 (§5.4)        |
| 자음/모음 확정 후  | 무효 (무시)                  |

### 5-8. FSM 차이점

공통 FSM(§1.7~§1.8)을 사용하되 다음을 추가한다:

1. **모음 순환 컨텍스트**: `3`/`6` 키 연속 입력 시 1회→2회 전환. 타임아웃 또는 다른 키 입력 시 확정.
2. **획추가 컨텍스트**: 조합 중인 자음 또는 모음에 대해 파생 테이블 적용.
3. **쌍자음 컨텍스트**: 조합 중인 자음에 대해 쌍자음 테이블 적용.

### 5-9. 검증 케이스

| 입력 시퀀스                | 기대 출력 | 설명                                     |
|----------------------------|-----------|------------------------------------------|
| `1` `3`                    | 가        | ㄱ + ㅏ(`3`×1)                           |
| `1` `3``3`                 | 거        | ㄱ + ㅓ(`3`×2)                           |
| `1` `3` `*`                | 갸        | ㄱ + ㅑ(ㅏ+`*`)                          |
| `1` `3``3` `*`             | 겨        | ㄱ + ㅕ(ㅓ+`*`)                          |
| `1` `6` `*`                | 교        | ㄱ + ㅛ(ㅗ+`*`)                          |
| `1` `6``6` `*`             | 규        | ㄱ + ㅠ(ㅜ+`*`)                          |
| `1` `*` `3`                | 카        | ㄱ→ㅋ(자음`*`) + ㅏ                      |
| `2` `*` `3`                | 다        | ㄴ→ㄷ(`*`) + ㅏ                          |
| `2` `*` `*` `3`            | 타        | ㄴ→ㄷ→ㅌ(`*``*`) + ㅏ                   |
| `8` `*` `3`                | 하        | ㅇ→ㅎ(`*`) + ㅏ                          |
| `1` `#` `3`                | 까        | ㄱ→ㄲ(`#`) + ㅏ                          |
| `7` `*` `#` `3`            | 짜        | ㅅ→ㅈ(`*`)→ㅉ(`#`) + ㅏ                  |
| `1` `6` `3`                | 과        | ㄱ + ㅘ(ㅗ+ㅏ, §1.5)                     |
| `8` `3` `9`                | 애        | ㅇ + ㅐ(ㅏ+ㅣ)                           |
| `8` `3` `*` `9`            | 얘        | ㅇ + ㅒ(ㅑ+ㅣ)                           |
| `1` `3` `2` `3``3`         | 간        | 가 + ㄴ(종성) — `3``3`은 ㅓ이므로 거 아님 |

## 6. SKY-II

### 6-1. 개요

팬택(구 SK텔레텍)의 SKY 폰에 탑재된 방식. 자음과 모음이 각 키에 2개씩 쌍으로 배치된 4×3 키패드이다.  
**별도의 Shift/쌍자음 키가 없으며**, 동일 키 반복 입력(멀티탭) 방식으로 모든 자모와 쌍자음을 입력한다.

- **1회**: 첫 번째 자모
- **2회**: 두 번째 자모
- **3회**: 쌍자음 (쌍자음이 있는 자음 키만 해당)

### 6-2. 키 레이아웃

```
[ㄱㅋ][ㅣㅡ][ㅏㅑ]
[ㄷㅌ][ㄴㄹ][ㅓㅕ]
[ㅁㅅ][ㅂㅍ][ㅗㅛ]
[ㅈㅊ][ㅇㅎ][ㅜㅠ]
```

| 키  | 1회 | 2회 | 3회(쌍자음) | 키 ID   |
|-----|-----|-----|-------------|---------|
| `1` | ㄱ  | ㅋ  | ㄲ          | 1행 1열 |
| `2` | ㅣ  | ㅡ  | (없음)      | 1행 2열 |
| `3` | ㅏ  | ㅑ  | (없음)      | 1행 3열 |
| `4` | ㄷ  | ㅌ  | ㄸ          | 2행 1열 |
| `5` | ㄴ  | ㄹ  | (없음)      | 2행 2열 |
| `6` | ㅓ  | ㅕ  | (없음)      | 2행 3열 |
| `7` | ㅁ  | ㅅ  | (없음)      | 3행 1열 |
| `8` | ㅂ  | ㅍ  | ㅃ          | 3행 2열 |
| `9` | ㅗ  | ㅛ  | (없음)      | 3행 3열 |
| `*` | ㅈ  | ㅊ  | ㅉ          | 4행 1열 |
| `0` | ㅇ  | ㅎ  | (없음)      | 4행 2열 |
| `#` | ㅜ  | ㅠ  | (없음)      | 4행 3열 |

### 6-3. 키 입력 규칙 (멀티탭)

| 상황                         | 처리                                              |
|------------------------------|---------------------------------------------------|
| 키 1회 입력                  | 해당 키의 1번째 자모 입력 (조합 대기)             |
| 동일 키 2회 입력              | 2번째 자모로 교체 (확정 아님)                     |
| 동일 키 3회 입력 (쌍자음 있음) | 쌍자음으로 교체                                  |
| 동일 키 3회 입력 (쌍자음 없음) | 1번째 자모로 순환                                |
| 다른 키 입력 / 타임아웃       | 현재 자모 확정 후 새 키 처리                      |

> **타임아웃**: 기본 1000ms.  
> 쌍자음(`ㄲ`,`ㄸ`,`ㅃ`,`ㅆ` — ㅅ의 쌍자음은 `7`키에 없음 주의, `ㅉ`)은 해당 키 3회 입력으로 생성된다.

### 6-4. 쌍자음 입력 경로

| 쌍자음 | 키  | 입력 횟수 |
|--------|-----|-----------|
| ㄲ     | `1` | 3회       |
| ㄸ     | `4` | 3회       |
| ㅃ     | `8` | 3회       |
| ㅆ     | — | ※ ㅅ이 `7`키 2회(ㅁ→ㅅ)에 위치하므로, ㅆ 입력은 별도 확인 필요 |
| ㅉ     | `*` | 3회       |

### 6-5. 복합 중성 조합

SKY-II는 모음 단독 키 입력으로 기본 모음을 입력하며, 복합 중성은 §1.5 테이블에 따라 두 모음을 연속 입력하여 생성한다.

| 입력 시퀀스 | 결과    | 설명            |
|-------------|---------|-----------------|
| `9`(ㅗ) `3`(ㅏ) | ㅘ  | ㅗ+ㅏ           |
| `9`(ㅗ) `2`(ㅣ) | ㅚ  | ㅗ+ㅣ           |
| `#`(ㅜ) `6`(ㅓ) | ㅝ  | ㅜ+ㅓ           |
| `2`(ㅡ) `2`(ㅡ) 2회→ `2`(ㅣ) | ㅢ | ㅡ+ㅣ (2키 2회=ㅡ 후 ㅣ 입력) |

### 6-6. FSM 차이점

공통 FSM(§1.7~§1.8)을 사용하되 다음을 추가한다:

1. **키 반복 컨텍스트**: 동일 키 연속 입력 횟수를 추적하여 1회/2회/순환 자모 결정. 타임아웃 또는 다른 키 입력 시 확정.
2. 그 외 FSM 전이는 공통 규칙과 동일.

### 6-7. 검증 케이스

| 입력 시퀀스              | 기대 출력 | 설명                          |
|--------------------------|-----------|-------------------------------|
| `1` `3`                  | 가        | ㄱ(1×1) + ㅏ(3×1)             |
| `1``1` `3`               | 카        | ㄱ→ㅋ(1×2) + ㅏ               |
| `1``1``1` `3`            | 까        | ㄱ→ㅋ→ㄲ(1×3) + ㅏ            |
| `4` `3`                  | 다        | ㄷ(4×1) + ㅏ                  |
| `4``4` `3`               | 타        | ㄷ→ㅌ(4×2) + ㅏ               |
| `4``4``4` `3`            | 따        | ㄷ→ㅌ→ㄸ(4×3) + ㅏ            |
| `5` `3`                  | 나        | ㄴ(5×1) + ㅏ                  |
| `5``5` `3`               | 라        | ㄴ→ㄹ(5×2) + ㅏ               |
| `5``5``5` `3`            | 나        | ㄴ→ㄹ→ㄴ 순환(5×3) + ㅏ       |
| `0` `3`                  | 아        | ㅇ(0×1) + ㅏ                  |
| `0``0` `3`               | 하        | ㅇ→ㅎ(0×2) + ㅏ               |
| `3``3` `3`               | 야        | ㅏ→ㅑ(3×2) + ㅏ? → ㅑ 확정 후 ㅏ |
| `1` `3``3`               | 갸        | ㄱ + ㅑ(3×2)                  |
| `0` `9` `3`              | 와        | ㅇ + ㅘ(ㅗ+ㅏ)                 |
| `0` `2``2` `3`           | 으아       | ㅇ+ㅡ(2×2) 확정 후 아 시작    |
| `1` `3` `5` → BS         | 가        | 종성 ㄴ 제거                  |
| `1` `3` → BS             | ㄱ (S1)   | 중성 제거                     |
| `1``1``1` `3` → BS       | ㄲ (S1)   | ㄲ 초성 상태 중성 제거        |

## 7. 모토로라 (Motorola)

### 7-1. 개요

모토로라 피처폰에 탑재된 독자 한글 입력 방식. 12키 기반이며 자음은 3~4개 키로 분산 배치하고, 모음은 `*`/`#` 조합을 사용한다. 제조사 모델별로 세부 차이가 존재하며, 여기서는 공통 방식을 정의한다.

### 7-2. 키 레이아웃

```
[ㄱㅋ][ㄴㅁ][ㅏㅓ]
[ㄷㅌ][ ㄹ ][ㅗㅜ]
[ㅂㅍ][ ㅅ ][ㅣㅡ]
[ㅈㅊ][ㅇㅎ][ 한 ]
```

| 키  | 1회 | 2회 | 키 ID  |
|-----|-----|-----|--------|
| `1` | ㄱ  | ㅋ  | 1행 1열 |
| `2` | ㄴ  | ㅁ  | 1행 2열 |
| `3` | ㅏ  | ㅓ  | 1행 3열 |
| `4` | ㄷ  | ㅌ  | 2행 1열 |
| `5` | ㄹ  | (없음) | 2행 2열 |
| `6` | ㅗ  | ㅜ  | 2행 3열 |
| `7` | ㅂ  | ㅍ  | 3행 1열 |
| `8` | ㅅ  | (없음) | 3행 2열 |
| `9` | ㅣ  | ㅡ  | 3행 3열 |
| `*` | ㅈ  | ㅊ  | 4행 1열 |
| `0` | ㅇ  | ㅎ  | 4행 2열 |
| `#` | 한/영 전환 | — | 4행 3열 |

> `ㄹ`(`5`), `ㅅ`(`8`)은 단독 자음으로 2회 입력 시 각각 독립 자음으로 처리된다.  
> `#`(한) 키는 한/영 모드 전환이다.

### 7-3. 키 입력 규칙

각 키의 **1번째 자모**는 해당 키만 누르면 입력된다.  
**2번째 자모**는 해당 키 입력 후 `[한]`(`#`) 키를 눌러 전환한다.

| 상황                             | 처리                                               |
|----------------------------------|----------------------------------------------------|
| 키 1회 입력                      | 해당 키의 1번째 자모 입력 (조합 대기)              |
| 키 입력 후 `[한]`(`#`) 입력      | 현재 자모를 같은 키의 2번째 자모로 교체            |
| 2번째 자모 상태에서 `[한]` 추가  | 다시 1번째 자모로 순환                             |
| 다른 키 입력 / 타임아웃           | 현재 자모 확정 후 새 키 처리                       |

> `[한]` 키(`#`)는 **직전에 입력되어 아직 확정되지 않은** 자모에만 작용한다.  
> 자모가 이미 확정된 후(예: 음절 완성 후) `[한]`을 누르면 무효 처리된다.  
> 이 방식은 lpcwiki 및 한국 특허 문서(KR101445061B1)에서 모토로라의 "변환키 방식"으로 기술되어 있으며, `#` 키가 Shift/변환키 역할을 한다고 명시되어 있다.

### 7-4. 쌍자음

별도의 쌍자음 키가 없다. 구현에 따라 동일 키 추가 입력 또는 별도 특수 키로 처리한다.

### 7-5. 복합 중성 조합

§1.5 테이블에 따라 두 모음을 연속 입력하여 생성한다.

| 입력 시퀀스          | 결과    | 설명        |
|----------------------|---------|-------------|
| `6`(ㅗ) `3`(ㅏ)      | ㅘ      | ㅗ+ㅏ       |
| `6`(ㅗ) `9`(ㅣ)      | ㅚ      | ㅗ+ㅣ       |
| `6``6`(ㅜ) `3``3`(ㅓ) | ㅝ    | ㅜ+ㅓ       |
| `9``9`(ㅡ) `9`(ㅣ)   | ㅢ      | ㅡ+ㅣ       |

### 7-6. FSM 차이점

공통 FSM(§1.7~§1.8)을 사용하되 다음을 추가한다:

1. **키 반복 컨텍스트**: 동일 키 연속 입력 횟수를 추적하여 1회/2회/순환 자모 결정. 타임아웃 또는 다른 키 입력 시 확정.
2. 복합 중성은 §1.5 테이블로 두 모음 연속 입력하여 생성 (SKY-II와 동일 방식).

### 7-7. 검증 케이스

| 입력 시퀀스                  | 기대 출력 | 설명                                       |
|------------------------------|-----------|--------------------------------------------|
| `1` `3`                      | 가        | ㄱ(1) + ㅏ(3×1회)                          |
| `1` `#` `3`                  | 카        | ㄱ→`[한]`→ㅋ + ㅏ                          |
| `2` `3`                      | 나        | ㄴ(2) + ㅏ                                 |
| `2` `#` `3`                  | 마        | ㄴ→`[한]`→ㅁ + ㅏ                          |
| `4` `3`                      | 다        | ㄷ(4) + ㅏ                                 |
| `4` `#` `3`                  | 타        | ㄷ→`[한]`→ㅌ + ㅏ                          |
| `0` `3`                      | 아        | ㅇ(0) + ㅏ                                 |
| `0` `#` `3`                  | 하        | ㅇ→`[한]`→ㅎ + ㅏ                          |
| `*` `3``3`                   | 저        | ㅈ(*) + ㅓ(3×2회)                          |
| `*` `#` `3`                  | 처        | ㅈ→`[한]`→ㅊ + ㅏ                          |
| `3``3`                       | ㅓ        | ㅏ(3×1회) → `[한]` 없이 다시 3 → 별개 글자 |
| `3` `#`                      | ㅓ        | ㅏ→`[한]`→ㅓ                               |
| `6` `3`                      | 과        | ㄱ + ㅘ(ㅗ+ㅏ, §1.5)                       |
| `6` `#` `3`                  | 궈        | ㄱ + ㅝ(ㅜ+ㅓ) — ㅗ→`[한]`→ㅜ 후 ㅓ       |
| `0` `6` `6` `#` `3` `#`      | 워        | ㅇ + ㅝ (ㅜ+ㅓ) — ㅗ→`[한]`→ㅜ, ㅏ→`[한]`→ㅓ |
| `1` `3` `2` → BS             | 가        | 종성 ㄴ 제거                               |
| `1` `3` → BS                 | ㄱ (S1)   | 중성 제거                                  |
| `1` `#` → BS                 | ㄱ (S1)   | ㅋ→BS→ㄱ (`[한]` 전 상태로 복원)           |

## 8. 공통 구현 지시

Claude Code는 아래 지시에 따라 구현하라.

### 8-1. 데이터 모델

```kotlin
/** 키 입력 추상화 */
sealed class KeyInput {
    data class Char(val key: kotlin.Char) : KeyInput()
    data class Special(val type: SpecialKey) : KeyInput()
}

enum class SpecialKey {
    BACKSPACE, SPACE, ENTER, CONFIRM,
    SHIFT, SHIFT_LOCK,
    DIRECTION_LEFT, DIRECTION_RIGHT, DIRECTION_UP, DIRECTION_DOWN,
    STROKE_ADD,   // * 키 (획추가/쌍자음)
    MODE_SWITCH   // 한영 전환
}

/** FSM 상태 */
enum class FSMState { S0, S1, S2, S3, S3D, S4 }

/** 현재 조합 중인 음절 상태 (immutable) */
data class SyllableState(
    val fsm: FSMState = FSMState.S0,
    val cho: Char? = null,       // 초성 자모
    val jung: Char? = null,      // 중성 자모 (복합 중성 포함)
    val jong: Char? = null,      // 종성 첫 번째 자모
    val jong2: Char? = null,     // 종성 두 번째 자모 (겹받침)
    // 레이아웃별 추가 컨텍스트
    val cycleKey: kotlin.Char? = null,  // 현재 순환 중인 키
    val cycleCount: Int = 0,            // 순환 횟수
    val vowelBuffer: List<kotlin.Char> = emptyList() // 모음 조합 버퍼 (천지인/SKY)
)

/** 키 입력 처리 결과 */
data class InputResult(
    val committed: String,       // 이번 입력으로 확정된 문자열
    val composing: String,       // 현재 조합 중인 문자 (화면 표시용)
    val newState: SyllableState
)
```

### 8-2. 핵심 함수 시그니처

```kotlin
/** 유니코드 음절 조합 */
fun compose(cho: Char, jung: Char, jong: Char? = null): Char

/** 유니코드 음절 분해 */
fun decompose(syllable: Char): Triple<Char, Char, Char?>

/** 복합 중성 조합 시도 */
fun compoundJungseong(first: Char, second: Char): Char?

/** 겹받침 조합 시도 */
fun compoundJongseong(first: Char, second: Char): Char?

/** 레이아웃별 키 입력 처리 (순수 함수) */
interface HangulAutomata {
    val layoutName: String
    fun process(state: SyllableState, input: KeyInput): InputResult
    fun flush(state: SyllableState): InputResult  // 강제 확정
    fun initialState(): SyllableState
}
```

### 8-3. 불변 구현 규칙

1. `SyllableState`는 반드시 `immutable`(val only). `process()`는 새 상태를 반환하며 부수효과 없음.
2. 모든 FSM 전이는 §1.8 규칙 표를 엄격하게 따른다.
3. `compose()` / `decompose()`는 §1.1~§1.4 공식만 사용하며 하드코딩 금지.
4. `flush()`는 커서 이동, 포커스 아웃, 한영 전환 등 외부 이벤트 시 반드시 호출된다.
5. 각 레이아웃은 `HangulAutomata` 인터페이스를 구현하고 레이아웃별 키 매핑/순환/모음 조합 로직만 캡슐화한다.

### 8-4. 레이아웃별 구현 클래스

## 9. 크로스 레이아웃 검증 케이스

동일한 단어를 각 레이아웃으로 입력했을 때 동일한 출력이 나와야 한다.

### 9-1. 검증 단어: "한글"

| 레이아웃    | 입력 시퀀스                                  | 기대 출력 |
|-------------|----------------------------------------------|-----------|
| 두벌식      | `G`(ㅎ) `K`(ㅏ) `S`(ㄴ) `R`(ㄱ) `L`(ㅣ)   | 한글      |
| 단모음      | `G`(ㅎ) `J`(ㅏ) `S`(ㄴ) `R`(ㄱ) `K`(ㅣ)   | 한글      |
| 천지인      | `8``8`(ㅎ) `ㅣ``ㆍ`(ㅏ) `5`(ㄴ) `4`(ㄱ) `1`(ㅣ) | 한글   |
| KT 나랏글   | `8`(ㅅ)`*``*``*`(ㅎ) `3`(ㅏ) `5`(ㄴ) `1`(ㄱ) `2``2`(ㅡ)`9`(ㅣ)=ㅢ? → `2`(ㅣ) | 한글 |
| SKY-II      | `0``0`(ㅎ) `3`(ㅏ) `5`(ㄴ) `1`(ㄱ) `2``2`(ㅡ→) `2`(ㅣ) | 한글   |
| 모토로라    | `0``0`(ㅎ) `3`(ㅏ) `2`(ㄴ) `1`(ㄱ) `9``9`(ㅡ)`9`(ㅣ)=ㅢ? → `9`(ㅣ) | 한글 |

### 9-2. 공통 Backspace 동작 검증

모든 레이아웃에서 아래 시퀀스는 동일하게 동작해야 한다:

| 단계 | 조작      | 화면 표시 | FSM 상태 |
|------|-----------|-----------|----------|
| 1    | ㄱ 입력   | ㄱ        | S1       |
| 2    | ㅏ 입력   | 가        | S2       |
| 3    | ㄴ 입력   | 간        | S3       |
| 4    | BS        | 가        | S2       |
| 5    | BS        | ㄱ        | S1       |
| 6    | BS        | (빈)      | S0       |
| 7    | BS        | (무시)    | S0       |

### 9-3. 겹받침 분리 검증 (공통)

모든 레이아웃에서 겹받침이 지원되는 경우 동일하게 동작해야 한다:

| 단계 | 조작    | 화면 표시 | FSM 상태  |
|------|---------|-----------|-----------|
| 1    | ㄷ 입력 | ㄷ        | S1        |
| 2    | ㅏ 입력 | 다        | S2        |
| 3    | ㄹ 입력 | 달        | S3        |
| 4    | ㄱ 입력 | 닭 (조합) | S3D       |
| 5    | ㅏ 입력 | 달가      | S2(새 글자) |
| 6    | BS      | 달        | S2(이전)  |

> 단모음/나랏글/모토로라 등 겹받침 입력이 우연히 발생할 수 있는 경우에도 §1.6 테이블에 따라 동일하게 처리한다.

### 9-4. 단모음 이중모음 Backspace 검증

단모음에서 이중모음(`J``J` → ㅑ 등)은 전처리 단계에서 FSM에 단일 모음으로 전달된다.  
따라서 Backspace는 §1.10 공통 규칙에 따라 **FSM 상태 기준으로 중성 단위로 삭제**한다.  
이중모음을 기본 모음으로 역분해하지 않는다.

| 단계 | 조작          | 화면 표시 | FSM 상태 | 설명                        |
|------|---------------|-----------|----------|-----------------------------|
| 1    | `J``J` (ㅑ)   | ㅑ        | S4       | 이중모음 전처리 후 FSM 진입 |
| 2    | BS            | (빈)      | S0       | 중성(ㅑ) 통째로 제거        |
| —    | —             | —         | —        |                             |
| 1    | `D` `J``J`    | 야        | S2       | ㅇ+ㅑ                       |
| 2    | BS            | ㅇ        | S1       | 중성(ㅑ) 통째로 제거        |
| 3    | BS            | (빈)      | S0       | 초성 제거                   |
