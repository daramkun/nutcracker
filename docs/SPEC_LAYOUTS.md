## 2. 두벌식 (Dubeolsik)

### 2-1. 키 레이아웃

#### Shift 적용

| 키     | 자모 | 키     | 자모 |
|--------|------|--------|------|

### 2-3. 자모 분류

```
자음(초성/종성 가능) = {ㄱ,ㄲ,ㄴ,ㄷ,ㄸ,ㄹ,ㅁ,ㅂ,ㅃ,ㅅ,ㅆ,ㅇ,ㅈ,ㅉ,ㅊ,ㅋ,ㅌ,ㅍ,ㅎ}

```

> 쌍자음(ㄸ, ㅃ, ㅉ)은 초성에만 사용 가능하며 종성에 올 수 없다.  
> ㄲ, ㅆ은 초성과 종성 모두 사용 가능하다.

### 2-4. FSM

두벌식은 §1.7 ~ §1.8의 공통 FSM을 **그대로** 사용한다. 레이아웃 고유 처리 없음.

| 입력 시퀀스 (키)        | 기대 출력   | 설명                          |
|-------------------------|-------------|-------------------------------|
| `r` `k`                 | 가          | 초성+중성                     |
| `r` `k` `s`             | 간          | 초성+중성+종성                |
| `r` `k` `s` `k`         | 가나         | 종성→다음초성 이동            |
| `e` `k` `f` `r`         | 닭 (조합중) | ㄷ+ㅏ+ㄹ+ㄱ = 겹받침 ㄺ      |

| `d` `h` `k`             | 와          | ㅗ+ㅏ 복합중성                |
| `d` `h` `o`             | 왜          | ㅗ+ㅐ = ㅙ                    |
| `r` `k` `s` → BS        | 가          | 종성 제거                     |
| `r` `k` → BS            | ㄱ (S1)     | 중성 제거                     |
| `r` → BS                | (빈)        | 초성 제거                     |
| `e` `k` `f` `r` → BS   | 달          | 겹받침 → 홑받침               |
| `e` `k` `f` `r` → BS → BS | 다       | 홑받침 제거                   |

## 3. 단모음 (Danmoeum)

### 3-1. 개요

단모음(Danmoeum) 레이아웃은 모음을 입력할 때 **같은 키를 두 번 누르는 방식**으로 겹모음을 입력한다.
- 첫 번째 입력: 단모음 (ㅏ, ㅑ, ㅓ, ㅕ, ㅗ, ㅡ)
- 두 번째 입력(타임아웃 내): 겹모음 (ㅐ, ㅒ, ㅔ, ㅖ, ㅙ, ㅚ, ㅢ)

### 3-2. 키 레이아웃

```
[ㅂ][ㅈ][ㄷ][ㄱ][ㅅ][ㅏ][ㅑ][ㅓ][ㅕ]
[ㅁ][ㄴ][ㅇ][ㄹ][ㅎ][ㅗ][ㅡ]
[⇧][ㅋ][ㅌ][ㅊ][ㅍ]
```

### 3-3. 키 매핑 테이블

#### 기본 (Shift 없음)

| 키 | 자모 | 키 | 자모 | 키 | 자모 |
|----|------|----|------|----|------|
| Q  | ㅂ   | Y  | ㅏ   | A  | ㅁ   |
| W  | ㅈ   | U  | ㅑ   | S  | ㄴ   |
| E  | ㄷ   | I  | ㅓ   | D  | ㅇ   |
| R  | ㄱ   | O  | ㅕ   | F  | ㄹ   |
| T  | ㅅ   | P  | ㅗ   | G  | ㅎ   |
| Z  | ㅋ   | H  | ㅡ   |    |      |
| X  | ㅌ   |    |      |    |      |
| C  | ㅊ   |    |      |    |      |
| V  | ㅍ   |    |      |    |      |

#### 중복 입력 (이중모음)

| 키     | 1회 | 2회 (타임아웃 내) |
|--------|-----|-------------------|
| Y      | ㅏ  | ㅐ               |
| U      | ㅑ  | ㅒ               |
| I      | ㅓ  | ㅔ               |
| O      | ㅕ  | ㅖ               |
| P      | ㅗ  | ㅙ (ㅗ+ㅏ)       |
| P × 2  | ㅗ  | ㅚ (ㅗ+ㅣ)       |
| H      | ㅡ  | ㅢ (ㅡ+ㅣ)       |

> Shift 조합은 두벌식과 동일하게 쌍자음 입력.

### 3-4. 자모 분류

두벌식과 동일 (§2-3 참조).

### 3-5. FSM - 단모음 전처리 컨텍스트

단모음 레이아웃은 **모음 입력 전처리**를 추가한다.

```
VowelContext:
  - lastVowelKey: 마지막 모음 키 (또는 null)
  - lastVowelTime: 마지막 모음 입력 시각
  - DOUBLE_TAP_TIMEOUT: 200ms (구현 시 조정 가능)

모음 입력 시 동작:
  1. (lastVowelKey == 현재키) AND (경과시간 < TIMEOUT) 
     → 이중모음으로 변환 후 FSM에 전달
  2. 아니면 → 단모음으로 FSM에 전달
  3. lastVowelKey = 현재키, lastVowelTime = now
```

**구체 예:**
- `Y` → {lastVowelKey=Y, current=ㅏ} → FSM(ㅏ)
- `Y`(200ms 내) → {lastVowelKey=Y가 이미 있음} → 이중모음 ㅐ 변환 → FSM(ㅐ)
- `Y` → `I`(200ms 내) → Y 타임아웃 미실행, I 단모음 → FSM(ㅓ)

### 3-6. FSM

이중모음 전처리 후 공통 FSM (§1.7 ~ §1.8) 그대로 사용.

### 3-7. 검증 케이스

| 입력 시퀀스 (키)        | 기대 출력   | 설명                          |
|-------------------------|-------------|-------------------------------|
| `r` `y`                 | 가          | 초성+단모음 ㅏ                |
| `r` `y` `y`(200ms 내)  | 개          | ㅏ → ㅐ 전환 후 FSM          |
| `r` `y`(200ms대기)      | 가          | 타임아웃 후 단모음 확정       |
| `r` `o` `o`(200ms 내)  | 과 (조합중) | ㅗ → ㅙ (ㅗ+ㅏ) 복합중성     |
| `r` `p` `p` `p`(각 200ms) | 고 → 과 | ㅗ → ㅙ → ㅚ 순차 입력      |
| `r` `y` `s`             | 간          | 초성+중성+종성                |
| `r` `y` `s` → BS        | 가          | 종성 제거                     |

## 4. 천지인 (Cheonjiin)

### 4-1. 개요

천지인 레이아웃은 **모음을 3개 기본키의 조합**으로 만드는 방식이다.
- 기본 3개 자모: **ㆍ**(중점, Sky), **ㆍ**(Down), **ㅣ**(Right)
- 조합으로 14개 모음 생성

### 4-2. 키 레이아웃

```
[ㅂ][ㅈ][ㄷ][ㄱ][ㅅ][ㅋ][ㅌ][ㅊ][ㅍ]
[ㅁ][ㄴ][ㅇ][ㄹ][ㅎ][1][2][3]
```

1번 키 = ㆍ (Sky, 중점)  
2번 키 = ㆍ (Down, 점)  
3번 키 = ㅣ (Right, 세로줄)

### 4-3. 모음 조합 테이블

| 조합                 | 결과 모음 | 조합                 | 결과 모음 |
|----------------------|----------|----------------------|----------|
| 1 (ㆍ)              | ㅏ       | 2 (ㆍ)              | ㅓ       |
| 3 (ㅣ)              | ㅣ       | 1+3 (ㆍ+ㅣ)        | ㅑ       |
| 2+3 (ㆍ+ㅣ)        | ㅕ       | 1+2 (ㆍ+ㆍ)        | ㅗ       |
| 1+2+3 (ㆍ+ㆍ+ㅣ)   | ㅠ       | 1 (ㆍ반복)          | ㅐ       |
| 2 (ㆍ반복)          | ㅔ       | 3 (ㅣ반복)          | ㅡ       |
| 1+2 (ㆍ+ㆍ반복)     | ㅘ       | 2+3 (ㆍ+ㅣ반복)    | ㅖ       |

> 정확한 규칙은 구현체 VowelContext에서 키 누름 순서 및 타이밍 기반 조합.

### 4-4. 자모 분류

두벌식과 동일 (§2-3 참조).

### 4-5. FSM - 천지인 모음 전처리

```
VowelContext (천지인):
  - pressedKeys: Set<{1,2,3}>
  - pressOrder: [1,2,3,...] 순서 기록
  - lastKeyTime: 마지막 키 입력 시각
  - COMBINATION_TIMEOUT: 300ms

모음 입력 시:
  1. 키 {1,2,3} 누름 → VowelContext에 추가
  2. 다른 키 {1,2,3 외} 입력 또는 타임아웃
     → VowelContext를 모음으로 해석 → FSM에 전달
     → VowelContext 초기화
  3. 자음 또는 모음 FSM 전달 후 처리는 공통 FSM 동일
```

### 4-6. FSM

천지인 모음 전처리 후 공통 FSM (§1.7 ~ §1.8) 그대로 사용.

### 4-7. 검증 케이스

| 입력 시퀀스 (키)  | 기대 출력   | 설명                          |
|-------------------|-------------|-------------------------------|
| `r` `1`           | 가          | 초성+모음(1=ㅏ)              |
| `r` `2`           | 거          | 초성+모음(2=ㅓ)              |
| `r` `1` `3`       | 겨          | 초성+모음(1+3=ㅑ)            |
| `r` `1` `2`       | 고          | 초성+모음(1+2=ㅗ)            |
| `r` `1` `2` `3`   | 궈          | 초성+모음(1+2+3=ㅠ)         |
| `r` `1` `s`       | 간          | 초성+중성+종성                |
| `r` `1` `s` → BS  | 가          | 종성 제거                     |

## 6. SKY-II

### 6-1. 개요

SKY-II는 **멀티탭(Multi-tap)** 방식 레이아웃이다. 같은 키를 여러 번 누르면 그 키에 할당된 여러 자모 중 하나가 선택된다.
각 키마다 1회~4회 타인에 따라 다른 자모 출력.

### 6-2. 키 레이아웃 및 멀티탭

```
키 1: ㄱ → ㄲ → ㄴ → ㄷ
키 2: ㄸ → ㄹ → ㅁ → ㅂ
키 3: ㅃ → ㅅ → ㅆ → ㅇ
키 4: ㅈ → ㅉ → ㅊ → ㅋ
키 5: ㅌ → ㅍ → ㅎ → (불사용)

키 6: ㅏ → ㅐ → ㅑ → ㅒ
키 7: ㅓ → ㅔ → ㅕ → ㅖ
키 8: ㅗ → ㅘ → ㅙ → ㅚ
키 9: ㅝ → ㅞ → ㅟ → ㅡ
키 0: ㅢ → ㅣ → (불사용) → (불사용)
```

### 6-3. 자모 분류

두벌식과 동일 (§2-3 참조).

### 6-4. FSM - SKY-II 멀티탭 전처리

```
MultiTapContext:
  - currentKey: 현재 눌린 키
  - tapCount: 누른 횟수
  - lastKeyTime: 마지막 입력 시각
  - MULTITAP_TIMEOUT: 500ms

같은 키 연속 입력 시:
  1. (currentKey == 새로운키) AND (경과시간 < TIMEOUT)
     → tapCount++ → 같은 상태 유지
  2. (currentKey != 새로운키) OR (경과시간 >= TIMEOUT)
     → 이전 키의 자모(tapCount 기반) FSM 전달
     → currentKey = 새로운키, tapCount = 1
  3. 자음 또는 모음 FSM 전달 후 처리는 공통 FSM 동일
```

### 6-5. FSM

SKY-II 멀티탭 전처리 후 공통 FSM (§1.7 ~ §1.8) 그대로 사용.

### 6-6. 검증 케이스

| 입력 시퀀스 (키)           | 기대 출력    | 설명                          |
|----------------------------|-------------|-------------------------------|
| `1`                        | ㄱ (S1)     | 초성 ㄱ                       |
| `1` `1`(500ms 내)         | ㄲ (S1)     | ㄱ → ㄲ                       |
| `1` `1` `1`(각 500ms)     | ㄴ (S1)     | ㄱ → ㄲ → ㄴ                 |
| `1` `6`(500ms 내)         | 가          | ㄱ 확정 + ㅏ 입력             |
| `1` `1` `6`                | 까          | ㄲ 확정 + ㅏ 입력             |
| `4` `8` `8`               | 조 (조합중) | ㅈ+ㅘ (ㅗ→ㅘ)               |
| `4` `8` `8` → BS          | 조 → 중     | 겹받침 분리                   |

## 7. 모토로라 (Motorola)

### 7-1. 개요

모토로라 레이아웃은 **변환키(#)**를 사용하는 방식이다. 초성/종성과 모음을 **#키로 전환**하며 입력한다.

### 7-2. 키 레이아웃

```
[ㄱ][ㄷ][ㅁ][ㅅ][ㅇ][1][2][3]
[ㄴ][ㄹ][ㅂ][ㅈ][ㅎ][4][5][6]
[ㅋ][ㅌ][ㅊ][ㅍ][*][7][8][9]
[0][#]

모드 전환: # 키
- 초성 모드 (기본)
- 중성 모드 (#누름)
- 종성 모드 (##누름)
```

### 7-3. 키 매핑 테이블

#### 초성 모드 (기본)

| 키 | 자모 | 키 | 자모 | 키 | 자모 | 키 | 자모 |
|----|------|----|------|----|------|----|----|
| 1  | ㄱ   | 2  | ㄷ   | 3  | ㅁ   | 4  | ㅅ  |
| 5  | ㅇ   | 6  | -    | 7  | -    | 8  | -   |
| 9  | ㅋ   | 0  | ㅌ   | *  | ㅊ   | #  | 모드전환 |

(추가) ㄴ, ㄹ, ㅂ, ㅈ, ㅎ, ㅍ 등은 추가 배치 또는 Shift 조합

#### 중성 모드 (#누름)

| 키 | 자모 | 키 | 자모 | 키 | 자모 | 키 | 자모 |
|----|------|----|------|----|------|----|----|
| 1  | ㅏ   | 2  | ㅑ   | 3  | ㅓ   | 4  | ㅕ  |
| 5  | ㅗ   | 6  | ㅡ   | 7  | ㅠ   | 8  | ㅜ  |

#### 종성 모드 (##누름)

초성 모드와 동일 자모 배치 (초성과 종성 구분).

### 7-4. 자모 분류

두벌식과 동일 (§2-3 참조).

### 7-5. FSM - 모토로라 모드 전환 전처리

```
MotorolaMode:
  - currentMode: CHO (초성) | JUNG (중성) | JONG (종성)
  - modeChangeCount: # 누른 횟수
  - lastHashTime: 마지막 # 입력 시각
  - HASH_TIMEOUT: 300ms

# 키 입력 시:
  1. (경과시간 < TIMEOUT) → modeChangeCount++
  2. (경과시간 >= TIMEOUT) → modeChangeCount = 1
  3. modeChangeCount % 3 → 모드 결정 (0:CHO, 1:JUNG, 2:JONG)

숫자/기타 키 입력 시:
  - currentMode 기반 자모 해석 → FSM 전달
  - FSM에서 자음/모음 판별하여 처리
```

### 7-6. FSM

모토로라 모드 전환 전처리 후 공통 FSM (§1.7 ~ §1.8) 그대로 사용.
